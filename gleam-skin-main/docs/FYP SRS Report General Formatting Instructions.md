# FYP SRS Report General Formatting Instructions

## Report Formatting
The following instructions apply for the formatting of the report:
*   **Font:** Times New Roman
*   **Font Size:** 12
*   **Line and Paragraph Spacing:** 1.5 with Added Space After Paragraph
*   **Text Alignment:** Justified
*   **Margins:** Left – 3 cm, Right/Top/Bottom – 2.5 cm

### Tables and Figures
All tables must have a number and a title in Times New Roman Font of size 10, italicized, which should be displayed **above** the table. All figures must have a number and a caption in Times New Roman Font of size 10, italicized, which should be displayed **below** the figure.

### Headings
*   **1 Chapter Heading:** Times New Roman, 16, Bold, 1st letter of each word capitalized
*   **1.1 Heading Level 1:** Times New Roman, 14, Bold, 1st letter of each word capitalized
*   **1.1.1 Heading Level 2:** Times New Roman, 12, Bold, 1st letter of each word capitalized
*   **1.1.1.1 Heading level 3:** Times New Roman, 12, Bold, 1st letter of only the first word capitalized

### References
Use **IEEE referencing style**. At the place in the document where you want to cite, use a number in square brackets `[_]`. In the references section, provide the full reference of the work. 
[IEEE Reference Guide](https://ieeeauthorcenter.ieee.org/wp-content/uploads/IEEE-Reference-Guide.pdf)

## Abstract

# Chapter 3: Analysis and Design
Include details about Software Process Model, Data Flow Diagram, Object Models, Use Cases, Block Diagrams, Flow-charts, or pseudo code etc.

### 3.1 Functional Requirements
Clearly define the functional requirements. Use bullet points or tables for clarity.
*   The system shall allow users to log in securely.
*   The system shall generate reports based on input data.

### 3.2 Non-Functional Requirements
Specify requirements such as performance, security, scalability, usability, reliability, and compliance with industrial standards.

### 3.3 System Models
*   **3.3.1 Data Flow Diagrams (DFD):** Provide detailed DFDs for the system processes.
*   **3.3.2 Use Case Diagrams:** Illustrate system interactions using use case diagrams.
*   **3.3.3 ER Diagram:** Show the database schema using an Entity-Relationship Diagram.
*   **3.3.4 Sequence and Activity Diagrams:** Visualize workflows and processes.

### 3.4 External Interface Requirements
*   **3.4.1 User Interfaces:** Describe the system’s user interfaces, focusing on usability.
*   **3.4.2 Hardware Interfaces:** List the hardware components and their specifications.
*   **3.4.3 Software Interfaces:** Describe the APIs, libraries, and third-party software used.
*   **3.4.4 Communication Interfaces:** Detail the communication protocols and formats employed.

### 3.5 Constraints and Limitations
Highlight real-world constraints such as cost, time, and technological barriers.

### 3.6 Assumptions and Dependencies
List assumptions about the system and dependencies.
*   **Assumption:** Internet connectivity will be stable.
*   **Dependency:** System relies on high-resolution video feeds.
